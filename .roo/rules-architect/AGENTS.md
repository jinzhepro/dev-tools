# AGENTS.md

This file provides guidance to agents when working with code in this repository.

## 项目架构规则 (仅非显而易见信息)

### 核心架构设计

- 基于 Next.js App Router 的多页面应用，但所有页面都是客户端渲染
- 工具组件采用插件化架构，通过配置文件动态注册
- 使用静态生成优化性能，所有工具页面在构建时预生成

### 组件架构模式

- 每个工具都是独立的 React 组件，遵循相同的 UI 结构模式
- 组件间无共享状态，每个工具完全独立运行
- 使用 shadcn/ui 组件库确保设计一致性

### 数据流架构

- 工具配置集中管理在 `src/data/tools.js`，作为单一数据源
- 动态路由通过 `componentMap` 实现配置驱动的组件渲染
- 无服务端状态管理，所有状态都在客户端组件内部

### 样式架构

- 基于 Tailwind CSS v4 的原子化 CSS 系统
- 使用 shadcn/ui 的设计系统和 CSS 变量
- 颜色系统配置为 neutral 基础色，支持主题切换

### 性能优化架构

- React 编译器自动优化组件渲染
- 静态生成所有工具页面，无需服务端渲染
- 代码分割通过动态导入实现，每个工具独立加载

### 扩展性设计

- 新工具只需添加组件和配置，无需修改路由代码
- 工具分类系统支持灵活的分类管理
- UI 组件库可独立升级，不影响业务逻辑
